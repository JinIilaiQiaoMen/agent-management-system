## 📋 测试用例

### 用例1: 用户注册
**目标**: 验证用户能够成功注册
**前置条件**: 访问注册页面
**步骤**:
1. 访问 /auth/register
2. 输入姓名
3. 输入邮箱
4. 输入手机号（选填）
5. 输入密码
6. 确认密码
7. 点击"下一步"（步骤1）
8. 点击"注册"
**预期结果**: 用户注册成功，跳转到登录页

### 用例2: 用户登录
**目标**: 验证用户能够成功登录
**前置条件**: 已注册用户
**步骤**:
1. 访问 /auth/signin
2. 选择"邮箱"标签页
3. 输入邮箱
4. 输入密码
5. 点击"登录"
**预期结果**: 用户登录成功，跳转到仪表盘

### 用例3: 提交圣旨
**目标**: 验证用户能够成功提交圣旨
**前置条件**: 已登录用户
**步骤**:
1. 访问 /san-sheng
2. 点击"提交圣旨"按钮
3. 输入圣旨内容："帮我分析一下ABC科技公司的客户背景"
4. 点击"提交"
**预期结果**: 圣旨提交成功，显示处理进度

### 用例4: 查看审计报告
**目标**: 验证用户能够查看审计报告
**前置条件**: 已完成流程
**步骤**:
1. 访问 /jinyiwei
2. 点击"审计报告"
3. 点击任意报告查看详情
**预期结果**: 显示完整的审计报告，包括5个维度指标

### 用例5: 升级订阅
**目标**: 验证用户能够升级订阅
**前置条件**: 已登录用户
**步骤**:
1. 访问 /subscriptions
2. 查看当前订阅
3. 点击"升级"按钮
4. 选择"专业版"
5. 确认升级
**预期结果**: 订阅升级成功，显示升级后的套餐信息
