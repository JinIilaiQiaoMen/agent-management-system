## 🎨 页面功能说明

宠物内容生成页面现在有**两个标签页**：

### 📝 标签 1: 文案生成
- 生成 TikTok、Instagram、YouTube、Amazon 等平台的文案
- 支持多语言、多平台、多版本生成

### 🖼️ 标签 2: ComfyUI 图像生成 (NEW!)
这是新增的功能，包含 4 个核心功能：

| 功能 | 图标 | 描述 |
|------|------|------|
| **场景图生成** | 🖼️ | 生成产品在真实场景中的渲染图 |
| **开箱图生成** | ⚙️ | 创建专业的开箱场景图片 |
| **卖点图生成** | ✨ | 突出产品核心卖点，生成展示图 |
| **短视频生成** | 🎬 | 自动生成 15-60 秒营销视频 |

## 🚀 如何使用 ComfyUI 功能

### 步骤 1: 进入页面

访问：http://localhost:5000/pet-content-generator

点击功能卡片后，系统会：
1. 调用 ComfyUI 服务

### 部署 ComfyUI 服务

**在使用 ComfyUI 功能前，必须先部署 ComfyUI 服务！**

#### 快速部署（5分钟）

```bash
# 1. 克隆 ComfyUI
git clone https://github.com/comfyanonymous/ComfyUI.git
cd ComfyUI

# 2. 安装依赖
pip install -r requirements.txt

# 3. 下载模型
# 访问 https://huggingface.co/stabilityai/stable-diffusion-xl-base-1.0
# 下载 sd_xl_base_1.0.safetensors 到 models/checkpoints/ 目录

# 4. 启动服务
python main.py --listen 0.0.0.0 --port 8188 --enable-cors-header "*"
```

## ❓ 常见问题

### Q1: 点击功能卡片没有反应？

**可能原因**：
1. 产品名称未填写
2. ComfyUI 服务未启动
3. 网络连接问题

**解决方法**：
1. 先在左侧表单填写产品名称和描述
2. 确认 ComfyUI 服务已启动（http://localhost:8188）
3. 查看浏览器控制台错误信息

### Q2: 提示 "ComfyUI 生成失败"？

**解决步骤**：
1. 检查 ComfyUI 服务是否运行：`curl http://localhost:8188`
2. 确认模型文件已下载：`ls ComfyUI/models/checkpoints/`
3. 检查环境变量配置：`.env.local` 中的 `COMFYUI_ENDPOINT`

### Q3: 生成的图片质量不好？

**优化方法**：
1. 使用更大的模型（如 SDXL）
2. 增加采样步数（20-50 步）
3. 使用更好的采样器（dpm++ 2m）
4. 添加更多产品细节描述

### Q4: 如何上传真实的产品图片？

**当前状态**：
- 演示模式使用占位符图片

**完整实现**：
- 需要添加图片上传功能
- 将上传的图片 URL 传递给 ComfyUI
- 参考 `/api/comfyui/test` 的参数说明
