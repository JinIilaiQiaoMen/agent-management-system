# 国内全渠道发布 - 问题修复说明

## 已修复的问题

### 1. 平台选择功能修复 ✅

**问题描述**：
- 点击"创建发布计划"后，选择平台时无法正确更新
- 发布时提示"平台 不支持内容类型: product"（平台名称为空）

**修复内容**：
- 在平台选择下拉框中添加了 `onChange` 事件处理器
- 当用户改变平台时，自动更新 `publishForm.platform`
- 同时根据新平台自动调整内容类型为该平台支持的第一个类型

**修复代码**：
```tsx
<select
  value={publishForm.platform}
  onChange={(e) => {
    const newPlatform = e.target.value;
    setPublishForm({
      ...publishForm,
      platform: newPlatform,
      contentType: DOMESTIC_PLATFORMS[newPlatform]?.supportedContentTypes[0] || 'product'
    });
  }}
>
```

### 2. 数据清理和验证 ✅

**问题描述**：
- 发布请求中包含空值或空字符串
- 可能导致后端处理错误

**修复内容**：
- 在发送请求前清理空值和空字符串
- 过滤掉空的 hashtag
- 只有内容类型为 'product' 时才发送 productInfo
- 添加了调试日志，方便排查问题

**修复代码**：
```tsx
const cleanForm = {
  ...publishForm,
  hashtags: publishForm.hashtags.filter(tag => tag.trim()),
  productInfo: publishForm.contentType === 'product' ? {
    price: publishForm.productInfo?.price || undefined,
    originalPrice: publishForm.productInfo?.originalPrice || undefined,
    stock: publishForm.productInfo?.stock || undefined,
    category: publishForm.productInfo?.category || undefined
  } : undefined
};
```

### 3. Hashtags 初始化修复 ✅

**问题描述**：
- hashtags 初始化为 `['']`（包含空字符串的数组）
- 可能导致数据处理问题

**修复内容**：
- 将 hashtags 初始化为空数组 `[]`
- 在提交时自动过滤空值

## 正确使用方法

### 方法 1：从平台卡片发布
1. 在平台列表中找到要发布的平台
2. 点击该平台卡片上的"发布"按钮
3. 模态框会自动选择该平台
4. 填写发布内容
5. （可选）设置定时发布
6. 点击"创建计划"

### 方法 2：从顶部按钮发布
1. 点击页面顶部的"创建发布计划"按钮
2. 在模态框中选择平台（默认是淘宝）
3. 根据选择平台，内容类型会自动调整
4. 填写发布内容
5. （可选）填写商品信息（仅电商）
6. （可选）设置定时发布
7. 点击"创建计划"

## 常见错误及解决方案

### 错误 1：缺少必要参数

**错误信息**：`发布失败: 缺少必要参数: platform, content`

**原因**：
- 平台未选择或选择无效
- 发布内容为空

**解决方案**：
1. 确保在模态框中选择了一个平台
2. 填写发布内容，不能为空

### 错误 2：不支持的平台

**错误信息**：`发布失败: 不支持的平台: xxx`

**原因**：
- 平台 ID 不在支持列表中

**解决方案**：
1. 从下拉框中选择有效的平台
2. 不要手动修改请求参数

### 错误 3：不支持的内容类型

**错误信息**：`发布失败: 平台 淘宝 不支持内容类型: video`

**原因**：
- 选择的内容类型不被该平台支持

**解决方案**：
1. 切换平台时，内容类型会自动调整
2. 手动选择该平台支持的内容类型
3. 查看平台卡片上的"支持的内容类型"标签

### 错误 4：未配置平台凭据

**错误信息**：`发布失败: 未配置平台凭据`

**原因**：
- 当前处于演示模式，没有配置真实的平台 API 凭据

**解决方案**：
1. 这是正常的，演示模式下所有发布都是模拟的
2. 如需真实发布，请配置环境变量（参考 `docs/domestic-platforms-guide.md`）
3. 演示模式下可以体验完整的发布流程

## 内容类型说明

不同平台支持的内容类型不同：

| 平台类型 | 支持的内容类型 |
|---------|----------------|
| 电商平台 | 商品、图片、视频、文章 |
| 社交媒体 | 文字、图片、视频、直播、短视频 |
| 内容平台 | 文章、视频、直播、短视频、音频 |
| 直播平台 | 直播、商品、视频 |

## 商品信息填写（仅电商平台）

当内容类型为"商品"时，可以填写以下信息：

- **价格**：商品销售价格
- **原价**：商品原价（用于显示折扣）
- **库存**：商品库存数量
- **分类**：商品所属分类

这些信息都是可选的，可以根据需要填写。

## 发布流程图

```

    ↓

    ↓

    ↓
填写发布信息
  ├─ 选择平台
  ├─ 选择内容类型
  ├─ 填写标题（商品）
  ├─ 填写内容
  ├─ 填写商品信息（电商）
  └─ 设置定时（可选）
    ↓
验证数据
    ↓
发送请求到后端
    ↓
后端处理
  ├─ 验证平台
  ├─ 验证内容类型
  ├─ 检查凭据
  └─ 执行发布/定时
    ↓
返回结果
    ↓
显示成功/失败提示
    ↓
关闭模态框，刷新列表
```
