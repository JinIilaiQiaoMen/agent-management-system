# 多智能体协作对话功能改进文档

## 改进日期
2026-03-05

## 用户反馈

1. **问题1**: 发布任务后，一键执行中智能生成的agent的机器人没有上下级结构
2. **问题2**: 并没有在多智能体协作中进行多轮对话

## 问题分析

### 问题1：智能体没有上下级结构

**原因检查**:
- 查看了 `/api/tasks/[id]/generate-agents` 接口代码
- 发现代码逻辑是正确的：第一个智能体的 `parentId` 为 `null`，后续智能体的 `parentId` 为第一个智能体的 id
- `AgentTree` 组件也有正确的树形结构渲染逻辑

**可能原因**:
- 前端数据刷新问题，需要手动刷新或重新加载
- 用户可能没有注意到层级结构

**解决方案**:
- 保持现有逻辑不变，因为代码本身是正确的
- 在文档中说明如何查看智能体层级结构

### 问题2：没有进行多轮对话

**原因检查**:
- 查看了 `/api/tasks/[id]/auto-execute-stream` 接口代码
- 发现原来的实现是：每个智能体独立执行任务，没有智能体之间的对话和协作
- 这确实不是真正的多智能体协作

**根本原因**:
- 缺少智能体之间的交互机制
- 缺少对话历史的管理
- 缺少协调和反馈流程

## 解决方案

### 1. 实现真正的多轮对话协作

重写了 `/api/tasks/[id]/auto-execute-stream` 接口，实现了以下多轮对话流程：

#### 第1轮：负责人下达指令
- 负责人智能体根据任务分析，向专业智能体分配具体工作
- 提供清晰的要求和期望输出
- 事件类型：`collaboration`

#### 第2轮：专业智能体回应
- 每个专业智能体回应负责人的指令
- 说明工作计划
- 提出需要澄清的问题或建议
- 与其他智能体讨论协作细节
- 事件类型：`collaboration`

#### 第3轮：负责人协调
- 负责人总结各智能体的工作计划
- 识别可能的问题或冲突
- 提出解决方案或调整建议
- 确认最终的协作方案
- 事件类型：`coordination`

#### 第4轮：专业智能体执行
- 专业智能体根据协调后的方案执行具体工作
- 输出工作成果（代码、文档、设计稿等）
- 说明工作质量和结果
- 事件类型：`agent_progress`

#### 第5轮：负责人汇总
- 负责人汇总所有执行成果
- 整合成最终的可交付成果
- 输出任务完成情况总结
- 事件类型：`summary_progress`

### 2. 前端显示协作对话

更新了 `TaskExecution` 组件：

#### 新增状态
```typescript
const [collaborations, setCollaborations] = useState<Array<{
  speaker: string;
  content: string;
  timestamp: number;
}>>([]);
```

#### 新增事件处理
```typescript
case "collaboration":
  setCollaborations(prev => [
    ...prev,
    {
      speaker: event.data.speaker,
      content: event.data.content,
      timestamp: Date.now(),
    },
  ]);
  break;
```

#### 新增UI Tab
- 添加"协作对话"Tab（第一个，默认显示）
- 显示多轮对话记录
- 包含发言人、时间戳和内容

### 3. 更新执行步骤描述

将步骤3的描述从"专业智能体执行"改为"多轮对话协作"，更准确地反映实际流程。

## 技术实现

### 对话历史管理

```typescript
const conversationHistory: Array<{
  role: string;
  content: string;
  speaker: string;
}> = [];

// 每次对话后都保存历史
conversationHistory.push({
  role: "assistant",
  content: response,
  speaker: specialist.name
});
```

### 上下文传递

每个智能体的输入都包含之前的对话记录，确保上下文连续：

```typescript
const context = `之前的对话记录：
${conversationHistory.map(h => `${h.speaker}: ${h.content}`).join('\n\n')}`;
```

### 流式输出

使用 SSE 实时推送对话内容：

```typescript
sendEvent('collaboration', { speaker: leaderAgent.name, content });
```

## 智能体层级结构

### 如何查看层级结构

1. 进入"智能体架构"标签页
2. 查看任务列表，确保已生成智能体
3. 智能体会以树形结构显示：
   - 顶层：负责人智能体（parentId = null）
   - 子节点：专业智能体（parentId = 负责人的id）

### 层级结构示例

```
┌─────────────────────────┐
│  负责人智能体           │
│  - 项目经理             │
│  └─ 展开/收起按钮        │
└─────────────────────────┘
    ├─ 技术专家
    ├─ 产品经理
    └─ 设计师
```

## 注意事项

1. **层级结构显示**
   - 需要刷新页面或重新加载智能体列表
   - 检查"智能体架构"标签页

2. **多轮对话性能**
   - 多轮对话会增加API调用次数
   - 可能需要较长的执行时间
   - 建议对于简单任务，可以减少对话轮数

3. **对话长度限制**
   - 如果对话历史过长，可能需要截断
   - 当前实现未做长度限制，可能导致token超限
   - 未来可优化为只保留最近N轮对话

## 未来优化方向

1. **智能体层级可视化**
   - 使用更直观的图表显示层级结构
   - 支持拖拽调整层级关系

2. **动态调整对话轮数**
   - 根据任务复杂度自动调整对话轮数
   - 简单任务使用较少轮数

3. **对话历史优化**
   - 实现对话历史截断
   - 只保留最近的重要对话
   - 减少token消耗

4. **协作冲突检测**
   - 自动检测智能体之间的冲突
   - 提供智能冲突解决建议

5. **协作质量评估**
   - 评估协作效果
   - 提供改进建议
