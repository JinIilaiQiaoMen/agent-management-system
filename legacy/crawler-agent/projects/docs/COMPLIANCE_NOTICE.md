# 合规使用提醒

⚠️ **重要提示：使用本工具前请仔细阅读本文档**

---

## 一、适用范围

本爬虫工具**仅适用于**：

✅ 合规的公开数据爬取  
✅ 学习研究用途  
✅ 已获得授权的数据获取  
✅ 遵守robots.txt协议的网站  
✅ 非商业性或已获商业授权的项目  

---

## 二、明确禁止的爬取场景

### 2.1 法律禁止的场景

❌ **个人信息爬取**
- 爬取姓名、电话、邮箱、身份证等个人信息
- 爬取社交媒体用户数据
- 爬取地址、银行卡号等敏感信息

❌ **商业机密数据**
- 爬取竞争对手的未公开商业数据
- 爬取付费内容、会员专享内容
- 爬取内部系统数据

❌ **版权保护内容**
- 未经授权爬取受版权保护的图片、视频、音乐
- 爬取付费电影、音乐、电子书
- 爬取新闻稿、学术论文等有版权的内容

❌ **恶意攻击行为**
- 使用爬虫进行DDoS攻击
- 恶意消耗服务器资源
- 破坏网站正常运行

### 2.2 道德禁止的场景

❌ **违反robots.txt**
- 明确禁止但仍然爬取
- 绕过robots.txt的限制

❌ **绕过反爬机制**
- 绕过验证码
- 破解加密保护
- 伪造登录凭证

❌ **不合理的访问频率**
- 高频访问造成服务器压力
- 无视Crawl-delay限制
- 并发请求过高

❌ **未经授权的商业用途**
- 将爬取数据用于商业竞争
- 转售爬取的数据
- 用于广告投放等商业行为

---

## 三、必须遵守的规则

### 3.1 robots.txt协议

**必须遵守robots.txt**，除非：

1. 目标网站没有robots.txt文件
2. robots.txt允许访问目标路径
3. 仅用于内部测试（使用`--no-robots`参数）

**如何遵守**：

```bash
# 检查robots.txt
curl https://example.com/robots.txt

# 爬虫工具会自动检查和遵守
# 如需跳过检查（仅测试用）
python web_crawler.py https://example.com --no-robots
```

### 3.2 访问频率控制

**必须控制访问频率**：

- 单线程模式：默认1-5秒随机延时
- 多线程模式：总并发不超过10
- 遵守`Crawl-delay`参数

**配置示例**：

```python
config = CrawlerConfig(
    delay_range=(2.0, 5.0),  # 2-5秒随机延时
    concurrency_mode='single',  # 单线程模式
    respect_rate_limit=True  # 遵守速率限制
)
```

### 3.3 数据使用规范

**爬取后的数据使用必须**：

1. **保留版权信息**
   - 不删除水印、版权标识
   - 注明数据来源

2. **不用于非法用途**
   - 不用于欺诈、诈骗
   - 不用于侵犯他人权益

3. **数据安全**
   - 妥善保管爬取的数据
   - 不泄露给第三方
   - 定期清理无用数据

### 5.1 可能面临的法律责任

违反相关法律法规可能面临：

- **民事责任**
  - 赔偿损失
  - 停止侵权
  - 消除影响

- **行政责任**
  - 罚款
  - 没收违法所得
  - 责令停业整顿

- **刑事责任**
  - 非法获取计算机信息系统数据罪
  - 侵犯公民个人信息罪
  - 侵犯著作权罪
  - 破坏计算机信息系统罪

### 5.2 相关法律条款

**《网络安全法》第二十七条**：
> 任何个人和组织不得从事非法侵入他人网络、干扰他人网络正常功能、窃取网络数据等危害网络安全的活动；不得提供专门用于从事危害网络安全活动的程序、工具。

**《个人信息保护法》第六条**：
> 处理个人信息应当具有明确、合理的目的，并应当与处理目的直接相关。采取对个人权益影响最小的方式。

**《著作权法》第五十三条**：
> 未经著作权人许可，复制、发行、表演、放映、广播、汇编、通过信息网络向公众传播其作品的，应当根据情况，承担停止侵害、消除影响、赔礼道歉、赔偿损失等民事责任。

## 六、合规使用建议

### 6.1 使用前检查清单

在爬取前，请确认：

- [ ] 已阅读并理解本合规提醒
- [ ] 已检查目标网站的robots.txt
- [ ] 已阅读目标网站的使用条款
- [ ] 确认爬取的数据为公开可访问的
- [ ] 确认不涉及个人信息或敏感数据
- [ ] 确认不涉及版权保护内容（或已获授权）
- [ ] 确认使用目的合法合规
- [ ] 如需商业用途，已获得授权
- [ ] 已设置合理的访问频率
- [ ] 已咨询法律专业人士（如不确定）

### 6.2 推荐做法

✅ **优先使用官方API**
- 许多网站提供官方API，这是最安全合规的方式
- 查看目标网站是否有开发者文档

✅ **联系网站获取授权**
- 商业用途务必提前联系
- 获取书面授权

✅ **使用代理和限速**
- 降低对目标网站的影响
- 避免被识别为异常访问

✅ **保留访问日志**
- 证明合规使用
- 便于问题追溯

❌ **避免的做法**
- 不绕过验证码
- 不伪造登录凭证
- 不破解加密保护
- 不恶意高频访问

## 八、遇到问题的处理

### 8.1 收到停止通知

如果收到目标网站的停止通知：

1. **立即停止爬取**
   - 停止所有正在进行的爬取任务
   - 删除已爬取的数据（如要求）

2. **与对方沟通**
   - 了解对方的具体要求
   - 说明爬取目的

3. **寻求法律帮助**
   - 咨询专业律师
   - 评估法律风险

### 8.2 疑似违规

如果不确定是否违规：

1. **立即停止**
   - 停止所有爬取活动
   - 保留相关日志

2. **咨询专业人士**
   - 咨询律师
   - 咨询技术专家

3. **删除敏感数据**
   - 删除可能违规的数据
   - 清除相关日志
