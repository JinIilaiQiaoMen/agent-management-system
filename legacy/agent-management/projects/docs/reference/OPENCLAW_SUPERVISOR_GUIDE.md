# OpenClaw 主管助手使用指南

## 一、简介

OpenClaw 主管助手是您的智能项目开发助手，通过自然语言对话，您可以完成以下工作：

- ✅ 创建、修改、删除文件
- ✅ 执行命令和脚本
- ✅ 调用项目 API
- ✅ 查询数据库
- ✅ 运行测试和构建
- ✅ 管理智能体和任务

## 二、快速开始

### 2.1 访问面板

访问 `/openclaw-supervisor` 页面，您将看到：

1. **项目状态** - 显示智能体数量、任务数量、构建状态
2. **快速操作** - 常用操作的快捷按钮
3. **对话区域** - 与 OpenClaw 交互的聊天界面
4. **指令说明** - 支持的指令格式

### 2.2 配置 OpenClaw

1. 访问 `/openclaw-integration` 页面
2. 填写 Webhook URL 和 API Key
3. 启用自动触发和通知
4. 点击"测试连接"验证配置

## 三、指令格式

### 3.1 代码操作

#### 创建文件
```
创建文件:src/app/new-page/page.tsx
```

OpenClaw 会：
1. 检查目录是否存在
2. 创建新文件
3. 添加默认内容

#### 读取文件
```
读取文件:src/app/page.tsx
```

OpenClaw 会：
1. 读取文件内容
2. 显示文件内容
3. 提供修改建议

#### 编辑文件
```
编辑文件:src/app/page.tsx
```

OpenClaw 会：
1. 读取当前内容
2. 询问修改内容
3. 执行修改
4. 验证修改结果

### 3.2 命令执行

#### 构建项目
```
执行命令:pnpm run build
```

或简化为：
```
构建:pnpm run build
```

#### 运行测试
```
执行命令:pnpm test
```

或简化为：
```
测试:pnpm test
```

#### 类型检查
```
执行命令:npx tsc --noEmit
```

### 3.3 数据查询

#### 查询智能体
```
查询:agents
```

返回所有智能体列表

#### 查询任务
```
查询:tasks
```

返回所有任务列表

#### 查询知识库
```
查询:knowledge-bases
```

返回所有知识库列表

### 3.4 API 调用

#### 创建智能体
```
创建智能体:名称:测试助手,角色:analyst,描述:这是一个测试智能体
```

#### 创建任务
```
创建任务:标题:分析数据,描述:分析销售数据,优先级:high
```

#### 查询任务状态
```
查询任务状态:task-id
```

## 四、使用场景

### 场景 1: 创建新页面

**用户输入**:
```
创建一个关于页面
```

**OpenClaw 处理**:
```
## 分析
用户需要创建关于页面

## 计划
1. 检查 src/app/about 目录是否存在
2. 创建 page.tsx 文件
3. 添加页面内容
4. 测试访问

## 执行
READ_FILE:src/app/about/page.tsx
CREATE_FILE:src/app/about/page.tsx:export default function AboutPage() { return <div>关于我们</div> }
EXECUTE:curl -I http://localhost:5000/about

## 结果
✅ 关于页面已创建
✅ 页面可以正常访问

## 建议
- 可以添加更多关于信息
- 考虑添加团队成员介绍
```

```
2. 检查删除 API 路由
3. 识别问题
4. 修复代码
5. 测试修复结果

EDIT_FILE:src/app/api/tasks/[id]/route.ts:...
```

## 五、高级功能

### 5.1 批量操作

可以一次性执行多个操作：

```
创建文件:src/app/api/test/route.ts
创建文件:src/app/api/test2/route.ts
执行命令:pnpm run build
```

### 5.2 条件执行

OpenClaw 会根据上下文判断是否需要执行某些操作：

```
检查文件是否存在:src/app/test/page.tsx
如果不存在则创建文件
```

### 5.3 错误恢复

当操作失败时，OpenClaw 会：
1. 分析错误原因
2. 提供修复建议
3. 自动重试（如果适用）
4. 记录错误日志

### 5.4 历史记录

所有操作都会被记录，可以随时查看：

```
查看历史记录
```

## 六、最佳实践

```
创建一个用户设置页面，包含：
- 用户名修改
```

✅ 好的示例：

```
测试新创建的页面
```

## 七、常见问题

### Q1: 如何撤销操作？

目前不支持撤销操作，但可以重新执行指令恢复。

### Q2: 如何查看执行日志？

所有操作都会记录在日志中，可以查看：

```
查看日志:最近10条
```

### Q3: 如何处理执行失败？

OpenClaw 会自动提供错误信息和修复建议。如果问题持续存在：
1. 检查输入格式
2. 查看详细错误信息
3. 联系技术支持

### Q4: 支持哪些操作？

支持以下操作：
- 文件操作（创建、读取、编辑、删除）
- 命令执行（shell 命令）
- API 调用（REST API）
- 数据查询（数据库）
- 测试和构建（npm 命令）

### Q5: 如何提升执行效率？

1. 使用批量操作
2. 提供具体的文件路径
3. 明确的操作描述
4. 合理使用缓存

## 八、限制和安全

### 8.1 操作限制

- 只能操作项目目录内的文件
- 不能执行破坏性命令
- 不能访问系统敏感文件
- 有操作频率限制

### 8.2 安全措施

- API Key 验证
- 操作白名单
- 输入验证
- 审计日志
